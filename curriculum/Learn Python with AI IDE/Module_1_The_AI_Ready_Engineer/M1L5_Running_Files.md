## 1. 脚本的本质：它只是纯文本

这是一个让很多初学者感到意外的事实：**一个所谓的“Python 脚本”，在物理本质上就是一个纯文本文档。**

它和你在电脑上写的 `.txt` 文档没有任何区别，里面装的都是字符。之所以叫它 `.py`，只是为了给操作系统看的一个“标签”，告诉电脑：“嘿，这个文件请交给 **Python 解释器 (Interpreter)** 来阅读并执行。”

> **🤔 什么是“解释器”？**
> 想象一下，你写的是字符（Python 代码），但电脑的 CPU 只听得懂机器语言（0 和 1）。**解释器**就像是一个全职的**“翻译官管家”**。当你运行脚本时，它会一行一行阅读你的文字，翻译成电脑能懂的指令并立刻执行。

### 🛠️ 实验：纯文本证明 (The Scribe's Trial)

我们将通过一个看似笨拙的实验，来打破对“脚本”的神秘感。

1.  **准备选址 (Callback L2)**：
    还记得 M1L2 创建的实验室吗？打开终端，确保你在“家目录” (`cd ~`)。如果你之前删掉了，请重新创建一个文件夹：
    ```bash
    mkdir my_ai_project
    cd my_ai_project
    pwd
    ```
    *   **提示**：执行 `pwd` 看到的这一串长长的路径就是这个文件夹在电脑里的“绝对地址”。你可以复制它，或者在 Finder (Mac) / 文件资源管理器 (Windows) 的地址栏里找到对应的位置。
2.  **创建文件**：
    保持终端开启。现在请打开你的图形化界面（Finder 或 文件资源管理器），进入刚才创建的 `my_ai_project` 文件夹。
3.  **系统编辑器登场**：
    *   **Apple Mac 用户**：打开你的 **TextEdit (文本编辑)** App。在菜单栏选择“文件” -> “新建”。务必确保它处于“纯文本模式”（如果看到标尺，请按 `Shift+Command+T` 转为纯文本）。
    *   **Windows 用户**：右键新建一个文本文档，或者打开电脑自带的 **记事本 (Notepad)**。
4.  **写入第一行“写作”代码**：
    在文档里写下这一行：
    ```python
    print("hello, written by text editor")
    ```
5.  **保存并变魔术：更名**：
    将文件保存到 `my_ai_project` 文件夹中，命名为 `hello.txt`。然后在文件夹中手动将其后缀名从 `.txt` 修改为 **`.py`**。
    *   *电脑可能会弹出安全警告，请大胆点击“确定”。*

## 3. 终端原生方式：touch 与 nano

虽然手动改后缀名能证明本质，但在工程开发中非常低效。身为一名工程师，你应该学会如何在终端里“一气呵成”地完成这一切。我们将创建第二个脚本。

### 3.1 制造空瓶：`touch` (Recall M1L2)

在终端中输入指令，制造第二个脚本的“容器”：
```bash
touch hello_again.py
```

### 3.2 编辑内容：`nano` (你的第一个终端编辑器)

我们要往这个空瓶子里装东西。输入以下指令进入一个简单的编辑界面：
```bash
nano hello_again.py
```
**操作指引**：
1.  在出现的黑色窗口中输入：`print("hello, written by nano editor")`。
2.  **保存并退出 (Survival Skill)**：
    *   按下 **`Ctrl + O`** (Write Out，保存)。
    *   按下 **回车键 (Enter)** 确认文件名。
    *   按下 **`Ctrl + X`** (Exit，退出)。

### 3.3 再次运行
```bash
python hello_again.py
```
**预期结果**：
打印出：`hello, written by nano editor`。

## 4. 殊途同归：正式定名为“脚本”

到这一步，你会发现无论是通过记事本改名，还是通过终端 `nano` 编写，它们**殊途同归**。

**现在，我们可以给这类以 `.py` 结尾的纯文本文件一个正式的工程称呼：Python 脚本 (Python Script)。**

*   **互通性**：你用 `nano` 写的文件，去图形化界面看，它也老老实实地呆在 `my_ai_project` 文件夹里。
*   **物理本质**：如果你现在把 `hello_again.py` 改名为 `hello_again.txt`，再去双击记事本打开，你会发现它依然是熟悉的文字。

**结论**：Python 并不神秘，它只是在执行一份“文本剧本”（脚本）而已。

## 5. 深度实验：消失的结果

在 M1L3 的 REPL 中，你输入 `1 + 1` 后回车，终端会立刻吐出 `2`。这是因为 REPL 的 **P** (Print) 会自动帮你把结果印出来。

**但在脚本模式下，Python 变“懒”了。**

请尝试创建一个新文件 **`nothing.py`**，把内容写成：
```python
1 + 1
2 ** 100
```
保存并运行 `python nothing.py`。你会发现：**终端一片漆黑，什么也没发生。**

**为什么？**
因为在脚本里，Python 确实默默算出了 `2` 和那串天文数字，但由于你没有下达“打印”指令，它算完就把结果悄悄丢掉了。

---

## 6. 总结：“对话” vs “写作”

通过上面的实验，我们可以清晰地对比出两种模式的差异：

| 特性 | REPL (交互模式 / L3) | Script (脚本模式 / L5) |
| :--- | :--- | :--- |
| **持久性** | 随用随弃 (口述) | 永久保存 (写作) |
| **反馈速度** | 即时反馈 (适合实验) | 运行前需保存 (适合生产) |
| **自动打印** | **自动显示结果** | **必须手动 `print()`** |
| **容错率** | 输错一行只能重来 | 可以随时修改并重复运行 |

---

## 7. 修正方法：召回消失的结果

你必须使用 Python 里的一个**“函数” (Function)** —— **`print()`**，明确命令 Python ：“把结果印给我看！”

> **💡 什么是函数 (Function)？**
> 暂时把它理解成电视遥控器上的**“功能按键”**。你按下一个键，电视就执行一个特定动作（比如静音）。`print()` 就是那个负责“把文字传送到屏幕上”的按键。关于它的详细原理，我们在 Module 2 会深入拆解。

```python
print(1 + 1)
print(2 ** 100)
```
重新运行，你找回了丢失的 `2` 和那个指数爆炸的结果。
