## 1. 理论：解释器的“即时对话” (Interactive Mode)

Python 是一种 **Interpreted (解释型)** 语言。这意味着它不需要像 C 或 Java 那样经过复杂的“编译”过程。你可以直接对着它说话，它立刻就会回应你。

这种交互方式被称为 **REPL**，它是程序员最亲密的“草稿纸”：
*   **Read (读取)**：读取你输入的代码。
*   **Eval (求值)**：计算这段代码的结果。
*   **Print (打印)**：在屏幕上输出结果。
*   **Loop (循环)**：等待你输入下一条指令。

在 AI 时代，REPL 是你验证 AI 生成的小片段是否可行的最佳战场。

## 2. 指令：唤醒 Python 之魂

由于我们在 M1_2 中已经通过 Conda 安装了 Python 3.12 并在终端激活了 `(ai_course)` 环境，现在召唤它非常简单。

### 2.1 启动解释器
在终端输入：
```bash
python
```
**观察变化**：
你会看到命令行提示符从 `$` 或 `>` 变成了 **`>>>`**。
只要你看到这三个大于号，就意味着你已经离开了系统的 Shell 层面，正式进入了 **Python 的精神世界**。

---

## 3. 对话：原子级的逻辑指令

在 `>>>` 后面，你可以开始下达指令。

### 3.1 基础算术 (Calculator Mode)
Python 是一个极佳的计算器。尝试输入并回车：
```python
1 + 1
100 / 3
2 ** 10  # 这是 2 的 10 次方
```

### 3.2 字符串与打印 (Text Output)
```python
print("Hello Antigravity")
"Hello" + " " + "World"  # 字符串拼接
```
> [!NOTE]
> 在 REPL 模式下，即使你不写 `print()`，直接输入一个值并回车，Python 也会把结果展示给你。但在正式的脚本文件里，你必须显式调用 `print()` 才能看到输出。

### 3.3 变量：给数据起名字 (Assignment)
变量是编程最核心的基石。在 REPL 中尝试：
```python
age = 25
name = "Leo"
age * 2
```
这时，`age` 这个名字在当前这个对话会话中就有了生命。即使你过几分钟再输入 `age`，解释器依然记得它是 25。这就是 **State (状态)**。

---

## 4. 机制：解释器背后发生了什么？

当你点击回车的那一刻：
1.  **词法分析**：Python 把你写的文字拆解成它认识的标记 (Tokens)。
2.  **句法验证**：它检查你是否少写了括号或打错了字。如果错了，它会立刻抛出一个 **SyntaxError (语法错误)**。
3.  **计算执行**：它调动 CPU 完成加减乘除，或者是分出一块内存来存储你的变量。

---

## 5. 退出咒语

当你玩够了，想回到 Shell 命令行时，你可以：
1.  输入 `exit()` 并回车。
2.  或者使用快捷键：**Mac 为 `Ctrl + D`**，**Windows 为 `Ctrl + Z` 然后回车**。

## 6. 硬核练习

1.  **变量漂移测试**：
    在 REPL 中依次执行：
    ```python
    x = 10
    y = x + 5
    x = 20
    ```
    请问此时 `y` 的值是多少？执行 `y` 回车看结果。
    **思考**：为什么 `x` 变了，`y` 却没有跟着变？理解“单次赋值”与“引用”的区别。

2.  **逻辑碰撞**：
    尝试在解释器里输入 `1 / 0`。
    观察弹出的报错信息。虽然这是个错误，但它也是一种“对话”。它告诉你为什么它无法执行你的意志。

3.  **数据类型初探**：
    尝试输入 `type(123)` 和 `type("123")`。
    看看 Python 是如何区分数字和文字的。
