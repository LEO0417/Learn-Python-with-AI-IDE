## 1. Philosophy: AI 不只是聊天機器人

很多初學者認為 AI 只是另一個搜尋引擎。但在 AI IDE 中，AI 的角色發生了質變。

### 1.1 從網頁對話到集成 Agent
在本課程體系中，我們所使用的 AI IDE 統一使用 **Antigravity**。它本身就是一個深度集成的 AI 開發環境。

*   **網頁端 (External AI)**：AI 無法感知你的檔案位置，也無法看清你的工程全貌。
*   **Antigravity (集成智能體功能)**：透過點擊界面上的 **Toggle Agent** 按鈕喚出。在這個 IDE 中，AI 與你共享同一個工作區，能感知你的檔案、終端機輸出甚至你當前操作的意圖。

### 1.2 理解協作的「邊界」
本課我們的重點是學會如何與 AI 溝通。我們不要求 AI 替我們寫複雜的程序，而是讓它作為一名「項目助理」，幫我們理解並管理現有的代碼。

---

## 2. The Raw Logic：上下文 (Context) 是溝通的命脈

### 2.1 為什麼 AI 會「一本正經地胡說八道」？
當 AI 給出錯誤回答時，90% 的原因是它缺乏**上下文 (Context)**。它不知道你的代碼存放在 `scripts` 資料夾裡，也不知道你已經創建了 `README.md`。

### 2.2 上下文的本質：索引與引用
在 **Antigravity** 中，我們可以透過以下方式賦予 Agent 「上帝視角」：
*   **檔案索引 (Indexing)**：IDE 會掃描整個項目，讓 Agent 對你的工作區佈局了然於胸。
*   **顯式引用 (@ 符號)**：在對話框輸入 `@`，可精準引用檔案、資料夾或終端機內容。

### 2.3 核心超能力：工具調用 (Tool Calling)
這是 Agent 區別於普通聊天機器人最強悍的地方。它不僅能動口，更能「動手」調用 IDE 的原生工具：
*   **終端機控制 (Terminal)**：AI 可以直接幫你輸入指令、執行 Python 腳本並查看報錯。
*   **檔案總管 (Explorer)**：AI 可以幫你新建、重命名或移動工程檔案。
*   **搜尋與跳轉**：在大項目中，AI 可以瞬間幫你找到特定的代碼邏輯。

## 3. Instruction：配置你的數位隊友

### 3.1 激活與模型手動選擇
1.  **召喚 Agent**：點擊界面上的 **Toggle Agent** 按鈕（或使用快捷鍵 `Cmd + E`）。
2.  **手動選擇模型**：在對話輸入框的下方，你會看到一個**模型選擇下拉框**。你可以根據需要手動切換不同的「大腦」：
    *   **Gemini** 或 **Claude**：它們擁有極強的邏輯推理能力，是處理工程複雜問題的首選。
    *   *注意：不同的模型在執行速度和理解深度上略有差異，你可以透過手動切換來對比效果。*

### 3.2 模式選擇：Fast 還是 Planning？
在 Toggle Agent 界面中，你還需要選擇 Agent 的執行模式：
*   **Fast (快速模式)**：響應極快。適合日常的代碼解釋、簡單代碼修改和基礎問答。**這是我們 Module 1 的首選默認模式。**
*   **Planning (規劃模式)**：Agent 會在執行前進行深度思考，甚至會拆解出完整的任務清單（Task List）。這種模式適合處理跨檔案的工程重構。
*   *在 Module 1 的練習中，我們暫時默認使用 **Fast** 模式。*

## 4. Practice：讓數位隊友分析你的「工作室」

我們將邀請 AI 助理來巡視我們在 M1L7 中整理好的項目。

### 4.1 開啟對話：分析檔案結構
在 AI Sidebar 對話框中輸入指令：
> 「請分析我的項目結構 @Folder。你認為我把代碼放在 `scripts` 資料夾，把說明放在 `README.md` 是否符合工程規範？」

### 4.2 深度對話：解釋代碼內容
1.  在左側檔案總管中，開啟 `scripts/ide_test.py` （或 `ide_demo.py`）。
2.  在對話框中嘗試輸入：
    > 「請幫我解釋一下 @ide_test.py 裡的代碼是在做什麼？」

### 4.3 觀察 AI 的洞察力
**預期結果**：AI 應該能認出你的檔案位置，並表揚你規範的項目組織方式，同時準確解釋出 `print` 語句的作用。
