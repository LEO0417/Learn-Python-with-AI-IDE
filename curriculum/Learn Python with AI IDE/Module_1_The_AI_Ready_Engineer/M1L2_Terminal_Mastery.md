## 1. 为什么工程师偏爱“黑框框”？

在图形界面 (GUI) 时代，我们习惯了用鼠标双击文件夹。
但在工程世界，鼠标太慢了。

*   **GUI (鼠标)**：像是步行。所见即所得，但难以批量操作（比如“把这1000个文件中含 'error' 的找出来”）。
*   **CLI (命令行)**：像是瞬间移动。你可以用一行代码对整个文件系统发号施令。

终端 (Terminal) 是你与计算机内核的直接对话窗口。掌握它，你就掌握了**对文件路径的绝对控制权**。

---

## 2. 认识你的 Shell

Shell 是你的翻译官。它接收你的文本指令，翻译给操作系统执行。
*   **macOS / Linux**: 默认使用 **Zsh**。
*   **Windows**: 默认使用 **PowerShell**。

这是一个**原生的环境**。我们现在不需要安装任何 Python 环境，只需要打开你电脑系统自带的终端。
*   Mac: `Command + 空格` -> 搜索 `Terminal` (终端)。
*   Windows: `Win` 键 -> 搜索 `PowerShell`。

---

## 3. 核心概念：路径的心理模型

这是新手最容易迷路的地方。无论你在做什么，永远在心里问自己：**“我现在在哪里？”**

### 3.1 绝对路径 (Absolute Path)
这是你的**GPS 坐标**。无论你身在何处，这个地址永远唯一。
*   **Mac**: `/Users/leowang/...` （以 `/` 开头）
*   **Win**: `C:\Users\leowang\...` （以盘符开头）

### 3.2 相对路径 (Relative Path)
这是你的**导航指令**（“向左转，再走10米”）。它完全依赖于你当前的起点。
*   `.` (点)：当前所在目录。
*   `..` (双点)：上一级目录（父目录）。
*   `./data`：去当前目录下的 data 文件夹。

> [!WARNING]
> **陷阱**：如果不清楚自己在哪里就使用相对路径，就像在陌生城市盲目右转，很容易掉进沟里（`FileNotFoundError`）。

## 4. 实战：导航指令集 (Navigation Suite)

请打开终端。我们开始演练。

### 4.1 罗盘：`pwd`
*   **全称**: Print Working Directory
*   **作用**: 告诉你现在的绝对坐标。
*   **工程意义**: 90% 的“找不到文件”错误，是因为你的 `pwd` 不在你以为的地方。

### 4.2 雷达：`ls`
*   **全称**: List
*   **作用**: 扫描当前目录下有什么。
*   **技巧**:
    *   `ls -a` (All)：查看隐藏文件（以 `.` 开头的文件，如系统配置）。
    *   `ls -l` (Long)：查看文件大小、权限等详细信息。

### 4.3 引擎：`cd`
*   **全称**: Change Directory
*   **作用**: 移动。
*   **终极技巧 (Home)**:
    *   **Mac**: `cd ~` (波浪号代表你的家目录 `/Users/yourname`)
    *   **Win**: `cd $HOME` (PowerShell 中) 或 `cd %USERPROFILE%` (CMD 中)
    *   **通用**: 直接输入 `cd` 回车，通常也会带你回家。

> [!TIP]
> **肌肉记忆：Tab 键**
> 永远不要完整打出文件名。输入前几个字母，按 **Tab** 键。
> 如果它没补全，说明要么你文件名打错了，要么你不在正确的目录下。**利用 Tab 键来验证你的路径直觉。**

## 5. 实战：创建与毁灭

### 5.1 创建：`mkdir`
*   `mkdir lab_space`: 创建文件夹。
*   `mkdir -p project/src/utils`: (Mac/Linux) 一次性创建多层目录。

### 5.2 毁灭：`rm`
> [!CAUTION]
> **危险指令**
> `rm file.txt` (Mac) 或 `Remove-Item file.txt` (Win)。
> **注意**：命令行删除的文件**不进回收站**。删了就是没了。
> 尽量少用 `rm -rf` (递归强制删除)，除非你 100% 确定路径是对的。

## 6. 工程自检 (Engineering Checkpoint)

### ✅ 验收标准 (Pass Criteria)
请完成以下连贯动作，验证你的方向感：

1.  **归零**：输入 `cd ~` (或者 `cd $HOME`) 回到家目录。运行 `pwd` 确认。
2.  **建立**：创建一个练习目录 `mkdir path_drill` 并进入 `cd path_drill`。
3.  **深潜**：创建子目录 `mkdir deep_folder` 并进入。
4.  **回溯**：使用相对路径 `cd ..` 回到 `path_drill`。
5.  **验证**：此时运行 `pwd`，路径结尾应该是 `.../path_drill`。

### 🛑 常见故障 (Fail & Triage)

*   **故障 1**：`cd: no such file or directory: Desktop`
    *   *原因*：如果你是中文系统，桌面可能叫 `桌面` 而不是 `Desktop`。或者你当前不在用户家目录下。
    *   *诊断*：先 `ls` 看看当前目录下到底有谁。
*   **故障 2**：Windows 下 `ls` 报错
    *   *原因*：如果在旧版 CMD 中，命令是 `dir`。但强烈建议使用 PowerShell，它兼容 `ls`。
*   **故障 3**：路径里有空格怎么办？
    *   *对策*：用引号包起来。`cd "My Project"`。

### 🧠 脑力小测 (Mental Model Quiz)
假设当前 `pwd` 是 `/A/B`，目录结构是 `/A/B/C/x.py`。
*   我想进入 C：`cd C` （对）
*   我想进入 C：`cd /C` （错！这是去根目录下的 C）
*   我想回到 A：`cd ..` （对）
