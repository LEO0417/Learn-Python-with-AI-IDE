## 1. 工程逻辑：目录是代码的“容器”

一个完整的**项目 (Project)** 除了代码，还可能包含数据、文档、配置以及依赖说明。

### 1.1 从单文件到目录树
如果把代码比作单一的实物，那么**项目目录**就是存放它们的“货架”。
*   **混乱布局**：所有文件（代码、日志、临时文件）全部堆在一起，难以维护且容易误删。
*   **规范布局**：优秀的开发者会通过子目录管理，将代码、数据与文档隔离。

### 1.2 为什么必须规范？
1.  **降低维护成本**：当你打开侧边栏时，清晰的目录结构能直观地展示工程的全貌。
2.  **提升协作效率**：在下一课中我们要引入 **AI 辅助能力**。结构清晰的项目能让 AI 通过文件名（如 `scripts` 或 `data`）迅速定位到相关背景，减少沟通偏差。

---

## 2. 理解映射：IDE 资源管理器与物理磁盘

IDE 左侧的**资源管理器 (Explorer)** 并不是一个独立的虚拟空间。它的物理本质是**本地磁盘文件夹在 IDE 中的“实时镜像”**。

### 2.1 实时同步逻辑
你可以进行以下验证：
1.  在 **Explorer (资源管理器)** 侧边栏的文件列表下方空白处点击右键，选择“新建文件夹”，命名为 `my_folder`。
2.  同时打开你的电脑系统窗口（Mac 的 Finder 或 Windows 的文件资源管理器），找到对应的物理路径。
3.  你会发现，文件夹在 IDE 里出现的瞬间，本地硬盘上也同步生成了。

**💡 效率贴士**：忘记侧边栏在哪了？试试 `Cmd + B` (Mac) 或 `Ctrl + B` (Windows) 来快速切换显示它。

**核心准则**：你在资源管理器里的每一次点击、创建或删除，都是对本地物理硬盘的真实操作。IDE 只是为你提供了一个更高效的**可视化管理界面**。

---

## 3. 实操指引：资源管理器的高效操作

### 3.1 核心操作集
在资源管理器侧边栏上，以下是开发者最常用的右键动作：
*   **新建文件 (New File)**：创建新的脚本或文本文档。
*   **新建文件夹 (New Folder)**：建立逻辑分类。
*   **在资源管理器中显示 (Reveal in Finder/Explorer)**：快速跳转到本地系统窗口。

### 3.2 项目名片：README.md
在软件工程中，`README.md` 是一个标准化的说明文档。
*   **用途**：记录项目功能、学习笔记以及运行方法。
*   **优势**：IDE 提供了对 Markdown 的原生支持，不仅可以编辑，还支持实时渲染预览。

---

## 4. 重点实验：重构项目工程

我们将把 L5 和 L6 的散装代码进行一次标准化的“搬迁”。

### 4.1 构建标准目录
在 IDE 的资源管理器中执行以下操作：
1.  **创建文件夹**：在根目录下创建一个文件夹 `scripts`。
2.  **移动资产**：用鼠标选中之前创建的 `ide_demo.py` 和 L5 的脚本文件，直接**拖拽**进 `scripts` 文件夹中。
    *   *说明*：当 IDE 询问“是否移动”时，点选确定。这会同步修改本地物理路径。
3.  **创建数据区**：创建一个名为 `data` 的空文件夹，用于后续课程存放外部数据。

### 4.2 编写项目说明
1.  在根目录下新建文件 `README.md`。
2.  输入一段标准化的项目描述：
    ```markdown
    # 我的第一个 Python 工程
    这是在 Module 1 创建的标准练习项目。
    
    - `scripts/`: 存放 Python 逻辑脚本。
    - `data/`: 存放相关数据文件。
    ```
3.  **预览验证**：点击编辑器右上角的“预览”按钮，查看排版效果。

## 5. 工程质量核验 (Engineering Checkpoint)

### ✅ 验收标准 (Pass Criteria)
1.  **目录层级**：根目录下仅包含 `scripts`、`data` 文件夹和 `README.md`。
2.  **同步验证**：在 IDE 内重命名文件夹，确认本地物理目录同步变更。
3.  **文档就绪**：`README.md` 包含初步的项目说明，并能正常渲染。
