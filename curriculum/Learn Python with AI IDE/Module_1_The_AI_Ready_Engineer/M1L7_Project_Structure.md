## 1. Philosophy: 项目是代码的“容器”

在 Module 1 的练习中，你可能已经创建了好几个 `.py` 文件。但在真正的软件开发中，一个完整的**项目 (Project)** 除了代码，还可能包含数据、文档和配置。

### 1.1 从单文件到文件夹
如果把代码比作“家具”，那么**项目文件夹**就是你的“房间”。
*   如果你把所有家具都堆在门口，你很难找到想要的东西。
*   优秀的工程师会在房间里摆放“柜子”（子文件夹），将代码和数据分类存放。

### 1.2 为什么必须整洁？
1.  **降低认知负担**：当你打开侧边栏，清晰的目录结构会告诉你“这里是代码，那里是数据”。
2.  **AI 友好的索引**：下一课我们要用的 **AI Agent** 其实是个“读图员”。如果你把项目整理得井井有条，AI 扫一眼文件夹名字（如 `scripts` 或 `docs`），就能立刻定位到它该去哪里辅助你。

## 2. The Raw Logic：所见即所得的同步

IDE 左侧的**资源管理器 (Explorer)** 并不是一个独立的虚拟空间。它的物理本质是**本地磁盘文件夹在 IDE 中的“实时镜像”**。

### 2.1 两个宇宙的同步
你可以做一个有趣的实验：
1.  在 IDE 的左侧空白处右键点击“新建文件夹”，命名为 `my_folder`。
2.  同时打开你的电脑窗口（Mac 的 Finder 或 Windows 的文件资源管理器），找到对应的物理路径。
3.  你会发现，文件夹在 IDE 里出现的瞬间，本地硬盘上也同步出现了。

**逻辑核心**：你在资源管理器里的每一次点击、拖拽和重命名，都是在直接操作你电脑里的硬盘。IDE 只是为你提供了一个更高效的**操作界面**。

---

## 3. Instruction：资源管理器的生存技巧

### 3.1 核心右键菜单
在资源管理器（侧边栏）上点击右键，你会看到几个最重要的工程动作：
*   **新建文件 (New File)**：不仅是 `.py`，以后我们还会创建文本、配置等。
*   **新建文件夹 (New Folder)**：建立分类。
*   **在资源管理器中显示 (Reveal in Finder/Explorer)**：如果你迷路了，这个功能会直接帮你打开本地电脑窗口。

### 3.2 档案的“名片”：README.md
在所有的规范项目中，都会有一个特殊文件：`README.md`。
*   **用途**：它是给人类阅读的“项目说明书”。
*   **优势**：IDE 对这种文件提供了完美的预览支持。你可以用它记录学习心得、运行方法。

## 4. Practice：整理你的“数字工作室”

让我们把 L5 和 L6 的散装练习进行一次正式的搬家。

### 4.1 手动构建工程蓝图
在 IDE 的资源管理器中，完成以下动作：
1.  **建柜子**：右键点击工程根目录，创建一个文件夹名为 `scripts`。
2.  **搬家**：用鼠标选中你之前创建的 `ide_demo.py` 和 L5 的那些小脚本，直接**拖拽**进 `scripts` 文件夹中。
    *   *观察*：IDE 可能会弹出窗口询问“是否要移动文件？”，大胆点击“确定”。
3.  **建数据区**：创建一个名为 `data` 的空文件夹，准备未来存放数据。

### 4.2 撰写第一份项目说明
1.  在根目录下新建文件 `README.md`。
2.  输入一段文字。例如：
    ```markdown
    # 我的第一个 Python 项目
    这是我在 Module 1 建立的学习区。包含以下内容：
    - `scripts/`: 学习 Python 基础操作的脚本。
    - `data/`: 未来存放练习数据的区域。
    ```
3.  **预览演示**：点击编辑器右上角的“预览”图标（通常是一个带放大镜的文档图标），感受 Markdown 文档带来的专业显示效果。

## 5. Engineering Checkpoint (工程自检)

### ✅ 验收标准 (Pass Criteria)
1.  **目录整洁**：根目录下只有 `scripts`、`data` 文件夹和 `README.md`，不再有散乱的 `.py` 文件。
2.  **同步验证**：在 IDE 里重命名一个文件夹，确认本地电脑对应的文件夹也跟着变了。
3.  **Markdown 预览**：能正常查看 `README.md` 的渲染效果。
