# Module 1 Lesson 3: The Ghost within the Shell (Python REPL)

> **Module 1: The Raw Interface**
> 目标：在黑框中召唤 Python 解释器，进行交互式对话。

---

## 1. 解释器 (Interpreter) 是什么？

上一节课我们学习了 **Shell**（壳），它是操作系统的翻译官。
现在我们需要另一个层面的翻译官 —— **Python Interpreter**。

计算机CPU只认识 0 和 1。Python 代码是写给人类看的。
解释器的作用，就是**实时**把你的 Python 语句翻译成机器指令并执行。

---

## 2. 召唤仪式

在终端中，输入以下命令并回车：

```bash
python
```
*(注：如果你的环境需要，可能要输入 `python3`)*

如果成功，你会看到两处明显的变化：
1.  **版本信息**：出现类似 `Python 3.12.x ...` 的文字。
2.  **提示符变化**：原来的 `$` 或 `>` 变成了三个大于号 `>>>`。

**`>>>` 标志着你已经离开了 Shell 的管辖，进入了 Python 的世界。** 这里也被称为 **REPL** (Read-Eval-Print Loop，读取-执行-打印-循环)。

---

## 3. Hello World 与 简单数学

在 `>>>` 后面，让我们试着说两句 Python 语。

### 3.1 打印 (Print)
输入可以被视为一种“输出行为”。告诉计算机把“Hello”显示在屏幕上。

```python
print("Hello World")
```
回车后，屏幕会单纯地显示 `Hello World`。

### 3.2 简单数学 (Math)
Python 是一个强大的计算器。

```python
1 + 1
```
回车后，显示 `2`。

```python
10 * 10
```
回车后，显示 `100`。

**思考**：在 Shell 里直接输 `1+1` 会怎样？
*   Shell 会报错，因为它不认识数学算式。
*   只有在 `>>>` 下，这才是合法的 Python 语法。

---

## 4. 逃离矩阵 (Exit)

如何回到普通的终端 Shell？
在 `>>>` 提示符下，输入：

```python
exit()
```
或者使用快捷键 `Ctrl + D` (Mac/Linux) / `Ctrl + Z` 然后回车 (Windows)。

看到提示符变回 `$` 或 `>`，你就回到了 Shell。

---

## 5. REPL 的局限性

既然我们可以这样写代码，为什么还需要文件？

*   **易失性**：你在 REPL 里写的 100 行代码，关掉窗口就全没了。
*   **不可复用**：无法保存，无法分享。

REPL 适合**快速验证**（比如算个数、测个函数），但要写软件，我们需要把咒语记录在卷轴上 —— 也就是 **.py 文件**。
