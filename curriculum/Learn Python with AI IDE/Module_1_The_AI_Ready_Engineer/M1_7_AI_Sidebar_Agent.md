# Module 1 Lesson 7: 智能助手 —— AI Sidebar Agent

> **Module 1: The AI-Ready Engineer**
> **目标**：认识你的超级合伙人。理解 AI 不只是对话框，而是能调动工具的 Agent。

---

## 1. 理论：从辅助工具到智能 Agent

在传统的 IDE 中，我们习惯于写代码、查文档、改错。这都是纯手工操作。
在 **Antigravity** 中，最大的革命性变化在于侧边栏那个名为 **Agent (智能体)** 的存在。

**什么是 Agent？**
它不仅仅是一个能聊天的 ChatGPT。它被赋予了**“行动力”**：
1.  **它能看**：它能阅读你整个工作区的文件内容。
2.  **它能动**：它能帮你创建文件、修改代码。
3.  **它能控**：它能直接向终端发送指令，帮你运行程序或安装库。

**这就是侧边栏。它是一个全能的工程合伙人。**

---

## 2. 机制：集成与调动 (Tool Use)

Agent 强大的秘密在于它与 IDE 深度打通。它可以调动一系列 **Tools**：

*   **Terminal Tool**：直接在底部终端执行命令。
*   **Search Tool**：在你的海量代码库中瞬间定位逻辑。
*   **File Tool**：全自动重写或新建文件。

> [!NOTE]
> **前沿预告**：
> 这种“Agent 调动工具”的模式，正是目前生成式 AI 最前沿的方向。在后续的 **Stanford CS146S** 课程中，我们将不仅是使用它，还会学习如何亲手构建这样的 Agent 系统。

---

## 3. 指令：调动你的 Agent

现在，请将目光移向侧边栏的 AI 对话框。尝试下达以下指令：

### 3.1 对话与询问
问它：`“帮我解释一下 Module 1 课程的目标是什么？”`
它会读取你项目中的文档（如 `M1_1_Why_Code.md`）并在几秒内给出总结。

### 3.2 指令模式
尝试输入：`“帮我在根目录下创建一个 test_agent.py 文件，里面写一个简单的加法函数，并运行它。”`
**观察：**
1.  它是否自动创建了文件？
2.  它是否自动在终端里运行了代码？
3.  它是否在执行前请求了你的确认？

---

## 4. 硬核练习

1.  **全能指令测试**：向 Agent 下达任务：`“清理我的工作区，删除刚才创建的所有带有 'test' 字样的文件。”` 观察它的判断力和执行过程。
2.  **报错共创练习**：故意在代码里写一个错误（比如 `print(x)` 但不定义 x），运行看到报错后，点击报错信息旁的“Ask AI”或直接在侧边栏问它：`“为什么这里报错了？”` 观察它如何结合报错堆栈给出修复建议。
