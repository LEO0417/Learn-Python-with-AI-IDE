## 1. 主流 IDE 与 AI 生态
在正式开始前，我们需要再次明确 IDE 的定义，并了解目前工程界主流的开发工具：

*   **IDE (Integrated Development Environment)**：集成开发环境。它不仅仅是一个写代码的文本编辑器，而是一个集成了文件管理、代码编辑、终端控制和调试功能的“军火库”。
*   **AI IDE**：在传统 IDE 的基础上，深度集成了 AI 能力的开发工具。
    *   **Antigravity**：我们课程使用的教学版 IDE。
    *   **Cursor**：业界首款流行的 AI 原生 IDE。
    *   **Windsurf**：由 Codeium 出品，其 Flow 功能提供了规范的 AI 协作编程方式。
    *   **Kiro**：新兴的 AI 原生 IDE，专注于极致性能与 AI 深度融合。

    *   **Trae**：字节跳动出品的 AI IDE。
*   **AI 增强插件**：即使是传统的 IDE (如 VS Code)，也可以通过安装插件获得 AI 辅助能力。这些插件包括：
    *   **Gemini CLI Companion**
    *   **Gemini Code Assist**
    *   **Claude Code for VS Code**
    *   **Codex – OpenAI’s coding agent**

## 2. 底层认知：界面布局与逻辑结构

### 2.1 界面布局 (The Interface Layout)
打开 Antigravity 后，我们需要将其调整为标准的工程开发布局。请熟悉以下三个核心功能区域：

1.  **资源管理器 (Explorer) - 左侧面板**
    *   **功能**：管理工程文件，浏览目录结构。
    *   **快捷键**：
        *   **Mac**: `Cmd + B`
        *   **Windows**: `Ctrl + B`

2.  **编辑器 (Editor) - 右上核心区**
    *   **功能**：编写代码、阅读逻辑的主要区域。
    *   **Pro Tip (视力友好)**：使用 `Cmd/Ctrl + +` 或 `-` 可以快速缩放整个界面的文字大小，找到最适合你的阅读比例。

3.  **终端面板 (Panel/Terminal) - 底部区域**
    *   **功能**：输入指令，查看解释器输出。
    *   **快捷键**：
        *   **Mac**: `Cmd + J`
        *   **Windows**: `Ctrl + J`

### 2.2 核心逻辑：主体与扩展 (Core & Extensions)

IDE 作为一个高度可定制的软件，其功能很大程度上依赖于它的**扩展 (Extensions)** 系统。

在左侧边栏点击“扩展”图标（四个方块）进入商店。对于 Python 开发，必须确保两个核心组件的安装：

1.  **Python 扩展**
    *   **作用**：赋予 IDE 识别、高亮、补全 Python 代码的能力，并负责关联后台解释器。
2.  **中文语言包 (Simplified Chinese)**
    *   **作用**：将界面语言本地化。

### 2.3 核心生存配置：告别低级错误

在正式编写代码前，我们需要调整两个能够大幅提升幸福感的“生存开关”：

1.  **开启自动保存 (Auto Save)**
    *   **操作步骤**：
        1.  呼出命令面板 (`Cmd/Ctrl + Shift + P`)。
        2.  输入并选择：`Preferences: Open Settings (UI)`。
        3.  在右侧搜索框输入：`files autosave`。
        4.  找到 **Files: Auto Save** 项，在下拉框中选择 **onFocusChange**。
    *   **优势**：当你点击“运行”按钮或切换到终端时，IDE 会自动保存。这既能保证运行的是最新代码，又不会因频繁保存而让项目的本地历史记录 (Timeline) 变得过于琐碎。
2.  **识别红色波浪线 (The Red Warning)**
    *   **认知**：当你输入错误的语法（如 `printt`）时，IDE 会立即在下方标注红色波浪线。
    *   **准则**：**不要带着波浪线运行代码**。这是 IDE 提前帮你识别出的“运行必崩”点。

> **说明**：左侧通常还提供 **AI 侧边栏 (Sidebar Agent)**。在本模块初期掌握基础工程能力时，建议先将其收起，专注于核心界面。

## 3. 实操指引：关联工程资产与环境

### 3.1 载入项目文件夹
将我们在 M1L5 中创建的脚本文件夹导入 IDE：
*   点击菜单栏 `文件 (File)` -> `打开文件夹 (Open Folder)`。
*   选中项目根目录。此时，左侧资源管理器会列出所有脚本。

### 3.2 配置解释器：环境映射 (Recall M1L4)

> **🔌 逻辑回调**
> 在 L4 创建的虚拟环境 `ai_course` 是代码运行的动力源。现在，我们需要将 IDE 映射到这个特定的环境。

**方式 A：通过命令面板 —— IDE 的“传送门” (推荐)**
作为专业开发者，你应该习惯通过快捷键呼出命令面板 (`Cmd/Ctrl + Shift + P`)。它像是一个直达任意配置的传送门，你不需要记住复杂的菜单路径，只需输入关键词即可：
1.  呼出命令面板。
2.  输入并选择 `Python: Select Interpreter`。
3.  在列表中选中带有 `ai_course` 标识的版本。

**方式 B：通过状态栏**
1.  打开任意 `.py` 文件。
2.  点击 IDE **右下角状态栏**的 Python 版本显示区。
3.  在弹出的列表中选择 `ai_course`。

*配置完成后，右下角应明确显示 `ai_course` 字样。*

## 4. 重点实验：从手动执行到集成运行

### 4.1 创建验证脚本
1.  **新建文件**：在左侧**资源管理器**中右键选择 `新建文件 (New File)`。
2.  **命名规范 (重要)**：将其命名为 `ide_demo.py`。
    *   **准则**：在工程领域，文件名应仅包含**小写字母、数字和下划线**。
    *   **禁忌**：**严禁使用空格**或**中文**命名代码文件，这会导致路径解析失败。
3.  **编写内容**：在编辑器中输入：
    ```python
    print("Hello AI IDE")
    ```
    *注意：此时你会发现代码出现了不同的颜色（高亮），这说明 Python 插件正在通过语法分析辅助你阅读。*

### 4.2 运行方式对比
*   **手动执行**：在底部的 IDE 终端中手动输入 `python ide_demo.py`。
*   **集成执行**：确认你要运行的文件处于**当前活动标签页 (Active Tab)**，点击编辑器右上角的**运行按钮**（三角形图标）。

**工程师的习惯：保持终端整洁**
当反复运行代码导致终端输出堆积如山时，可以在终端输入 `clear` (Mac) 或 `cls` (Windows) 并按回车。这能清空旧输出，让你专注于当下的运行结果。
