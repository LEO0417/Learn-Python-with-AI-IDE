## 1. Philosophy/Why：工程开发的“终极战场”

在之前的课程中，我们像原始人一样在黑色的荒野（终端）里徒手打磨石器：
*   **M1L2**：学会了如何在迷宫般的**路径**中穿行。
*   **M1L3/L5**：学会了**交互 (REPL)** 与**创作 (Script)** 的区别。
*   **M1L4**：在复杂的系统中开辟了专属于我们的**环境 (Environment)**。

虽然你已经具备了让代码跑起来的能力，但你一定感受到了那种“原始感”：没有语法高亮，没有自动补全，运行一个小程序需要在不同窗口间反复横跳。

**M1L6 是一个关键的转折点。** 我们将正式入驻 **IDE (Integrated Development Environment，集成开发环境)**。它不是一个全新的概念，而是将你之前学到的所有散落的技能（终端、解释器、文件系统）整合进一个统一的、高度自动化的驾驶舱中。

从这一刻起，这里将成为你未来学习 Python、构建 AI 应用以及协同 AI Agent 工作的唯一主战场。

## 2. The Raw Logic：IDE 的“三位一体”架构

我们使用的 IDE 名为 **Antigravity**。它的物理本质并不神秘，它的核心是将三个核心生产力工具“缝合”到了同一个界面下：

1.  **File Explorer (资源管理器)**：你的文件地图。取代了 `ls` 和 `cd`。
2.  **Code Editor (智能编辑器)**：你的写作台。它比记事本多出的功能叫做 **IntelliSense (智能感知)** 和 **Linting (实时纠错)**。
3.  **Integrated Terminal (集成终端)**：你的控制台。它就是你之前一直在用的 Shell。

**关键认知**：即使在 IDE 里，底层的物理规则依然没变。点击“运行”按钮的本质，依然是 IDE 悄悄在底部的终端里帮你打了一行 `python xxx.py`。**IDE 只是帮你省掉了那些机械性的重复劳动，而不是替代了原理。**

## 3. Instruction：核心机制与配置

### 3.1 万能入口：Command Palette (命令面板)

在专业工程师的世界里，鼠标是效率的敌人。掌握**命令面板**是区分专业开发者与业余爱好者的标志。

*   **唤醒指令**：
    *   **Mac**: `Cmd + Shift + P`
    *   **Windows**: `Ctrl + Shift + P`
*   **逻辑**：这是一个“全量搜索框”。无论是切换主题、查找文件，还是配置 Python，你只需要在这里输入关键词，它就会执行相应的操作。

### 3.2 配置解释器：插座与插头 (Recall M1L4)

这是本课最重要的配置步骤。IDE 必须知道应该用哪个 Python 来执行你的代码。

> **🔌 插座与插头隐喻**
> 在 M1L4 中，我们创建了虚拟环境 `ai_course`。你可以把它想象成一个墙上的**专用电源插座**，里面提供了特定的电压和工具。
> 现在的 IDE 就像一台昂贵的精密机器。点击运行之前，你必须选择正确的**插头 (Interpreter)**。

1.  唤出**命令面板**。
2.  搜索并选择：`Python: Select Interpreter`。
3.  在出现的列表中，**必须**选择带有 `ai_course` 标识的一项。
    *   *观察*：IDE 界面右下角的状态栏此时应显示 `3.13.x ('ai_course')`。如果不是，说明你的设备还没通电。

## 4. Practice：复刻与验证

我们将通过在 IDE 中重跑 M1L5 的逻辑，来验证整个工程链路的通畅。

### 4.1 打开文件夹 (Open Folder)
**工程准则**：不要仅打开单个代码文件，一定要打开该文件所属的**根文件夹**。这会让 IDE 建立起整个项目的索引。
*   `File -> Open Folder` -> 选择你在 M1L2 中创建的 `my_ai_project`。

### 4.2 创建演示脚本
1.  在左侧文件浏览器中右键选择 `New File`，命名为 `ide_test.py`。
2.  写入以下代码（体验**智能补全**，当你输入 `pri` 时，IDE 应该会提示 `print`）：
    ```python
    import sys
    from pathlib import Path

    print("--- 解释器链接验证 ---")
    # 打印当前正在运行代码的 Python 解释器路径
    print(f"当前使用的插座地址: {sys.executable}")
    
    # 回调 L2 的路径概念
    print(f"当前文件的物理锚点: {Path(__file__).resolve()}")
    ```

### 4.3 运行并透视真相
点击 IDE 右上角的绿色三角形（播放按钮）。

**硬核观察**：观察底部的 **Terminal** 窗口。你会发现它自动出现了一长串命令。那就是 IDE 帮你自动输入并执行的终端指令。

---

## 5. Engineering Checkpoint (工程自检)

### ✅ 验收标准 (Pass Criteria)
1.  **路径验证**：`ide_test.py` 输出的解释器路径必须包含 `envs/ai_course` (Mac/Linux) 或 `envs\ai_course` (Windows)。
2.  **状态栏验证**：右下角的 Python 环境提示准确显示了 `ai_course`。
3.  **联动验证**：在 IDE 内部的终端输入 `pwd`，显示的路径应正好是你打开的文件夹路径。

### 🛑 常见故障处理
*   **没高亮（全白）**：确认文件名后缀是 `.py` 而不是 `.txt`。
*   **ImportError**：通常是因为“插头”拔错了，请重新执行 `Select Interpreter`。
