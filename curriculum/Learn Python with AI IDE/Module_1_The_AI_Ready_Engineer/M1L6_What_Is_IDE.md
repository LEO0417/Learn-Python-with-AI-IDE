## 1. 主流 IDE 与 AI 生态
在正式开始前，我们需要再次明确 IDE 的定义，并了解目前工程界主流的开发工具：

*   **IDE (Integrated Development Environment)**：集成开发环境。它不仅仅是一个写代码的文本编辑器，而是一个集成了文件管理、代码编辑、终端控制和调试功能的“军火库”。
*   **AI IDE**：在传统 IDE 的基础上，深度集成了 AI 能力的开发工具。
    *   **Antigravity**：我们课程使用的教学版 IDE。
    *   **Cursor**：业界首款流行的 AI 原生 IDE。
    *   **Windsurf**：由 Codeium 出品，其 Flow 功能提供了规范的 AI 协作编程方式。
    *   **Kilo**：新兴的 AI 原生 IDE，专注于极致性能与 AI 深度融合。

    *   **Trae**：字节跳动出品的 AI IDE。
*   **AI 增强插件**：即使是传统的 IDE (如 VS Code)，也可以通过安装插件获得 AI 辅助能力。这些插件包括：
    *   **Gemini CLI Companion**
    *   **Gemini Code Assist**
    *   **Claude Code for VS Code**
    *   **Codex – OpenAI’s coding agent**

## 2. The Raw Logic：界面感知与“生存配置”

### 2.1 界面初始化 (The Initial Setup)
打开 Antigravity 后，我们需要将界面调整为最经典的“工程模式”。请熟悉以下三个核心区域及其快捷键区分：

1.  **资源管理器 (Explorer) - 左侧栏**
    *   **功能**：管理工程资产，浏览文件结构。
    *   **快捷键**：
        *   **Mac**: `Cmd + Shift + E`
        *   **Windows**: `Ctrl + Shift + E`

2.  **编辑器 (Editor) - 右上核心区**
    *   **功能**：编写代码的主战场。
    *   **快捷键**：`Cmd/Ctrl + 1` (快速切回第一个编辑窗口)

3.  **终端面板 (Panel/Terminal) - 底部控制区**
    *   **功能**：执行指令，查看输出。
    *   **快捷键**：
        *   **Mac**: `Cmd + J` 或 `Ctrl + ~`
        *   **Windows**: `Ctrl + J` 或 `Ctrl + ~`

此外，左侧通常还有一个 **AI 侧边栏 (Sidebar Agent)**，暂时将其**隐藏**，我们需要专注于核心工程界面。

现在，你应该看到一个稳定的布局：**左侧资源管理器**、**右上编辑区**、**下部终端区**。

### 2.2 核心逻辑：宿主与扩展 (Host & Extensions)

IDE 就像一个精密的硬件支架（宿主），但这仅仅是躯壳。**扩展 (Extensions)** 才是为它注入灵魂的功能模块。

在左侧边栏，点击那个“四个方块”组成的图标，这就是**扩展商店**。作为 Python 工程师，我们必须手动安装两个“灵魂”：

1.  **Python 扩展**
    *   **作用**：赋予 IDE 识别 Python 语言的能力。没有它，IDE 就只会被当成普通的记事本。它负责代码高亮、智能提示、以及代码运行。
2.  **中文语言包 (Simplified Chinese)**
    *   **作用**：将界面语言从英文切换为中文简体。安装后通常需要重启 IDE 生效。

*除此之外，资源管理器 (Explorer) 是我们最常驻的左侧面板，用于管理所有文件。*

## 3. Instruction：关联环境与项目

### 3.1 搬入项目文件夹
将我们在 M1L5 中辛苦打磨的“第一批移民”搬进 IDE。这也是我们在 L5 手动创建的那三个 `.py` 脚本文件（`hello.py` 等）。
*   点击菜单栏 `文件 (File)` -> `打开文件夹 (Open Folder)`。
*   选择该文件夹。此时，你可以在左侧看到所有的文件。

### 3.2 配置解释器：插座与插头 (Recall M1L4)
> **🔌 隐喻回调**
> 环境 `ai_course` 是我们的**专用电源插座**。现在，我们要为 IDE 这台设备接通电源。

我们有两种方式来完成这次“通电”：

**方式 A：通过命令面板 (标准做法)**
1.  呼出**命令面板** (`Cmd/Ctrl + Shift + P`)。
2.  搜索并选择 `Python: Select Interpreter` (选择解释器)。
3.  在列表中选中带有 `ai_course` 标识的一项。

**方式 B：通过状态栏 (快捷做法)**
1.  打开任意一个 `.py` 文件（例如 `hello.py`）。
2.  看向 IDE 的**右下角状态栏**，你会看到显示当前 Python 版本的地方（或者显示 `Select Interpreter`）。
3.  直接点击它，这会弹出和方式 A 一样的选择列表。
4.  选择 `ai_course`。

*操作完成后，请确认右下角已显示 `ai_course` 字样。*

## 4. Practice：复刻与验证

### 4.1 创建 IDE 演示脚本
1.  **创建文件**：在左侧**资源管理器**中右键点击文件夹空白处，选择 `新建文件 (New File)`，命名为 `ide_demo.py`。
2.  **进入编辑状态**：单击刚创建的 `ide_demo.py`，你会在右上的**编辑区**看到一个空白的输入窗口。
3.  **编写代码**：就像在记事本中输入文字一样，在窗口中输入：
    ```python
    print("Hello AI IDE")
    ```
    *你会发现：输入 `pri` 时会出现提示补全。这其实是 IDE 安装了 Python 插件后的“词典映射”功能，它能认出语法。未来我们加装 AI 引擎后，它将不再只是认词，而是能够理解你的意图。*

### 4.2 运行对比：手打 vs 一键
*   **方法 A (传统终端)**：在 IDE 底部的**终端**里，手动输入 `python ide_demo.py`。
*   **方法 B (IDE 一键运行)**：点击编辑器右上角的**运行按钮**（三角形图标）。

**硬核观察**：点击运行按钮后，请死死盯着底部终端。你会发现 IDE 帮你输入了一行极长的指令。
**结论**：IDE 并不是魔法，它是一个**集成的自动化平台**。除了运行代码，未来你还将在在这里管理数据库、管理 Git 仓库。它是你作为工程师的**唯一完整工作站**。
