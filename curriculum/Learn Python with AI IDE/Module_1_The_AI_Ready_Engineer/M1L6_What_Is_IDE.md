---

## 1. 为什么需要 IDE？

在 M1L5 中，我们虽然能跑代码，但效率极低：没有高亮、没有补全、没有错误提示。
**IDE (Integrated Development Environment)** 的本质，就是把软件开发所需的三个核心工具“缝合”到了同一个窗口里：

1.  **File Explorer (资源管理器)**：你的文件地图。
2.  **Code Editor (编辑器)**：你的写作台。
3.  **Terminal (终端)**：你的控制台。

我们使用的是 **Antigravity**（基于 VS Code 构建的 AI Native IDE）。

---

## 2. 界面解构：三个核心区域

请打开你的 IDE，我们一一对应。

### 2.1 左侧：资源管理器 (Explorer)
这是你管理文件的地方。
*   **功能**：新建、重命名、拖拽文件。
*   **优势**：你不需要再敲 `ls` 或 `mv` 命令，所见即所得。

### 2.2 中间：智能编辑器 (Editor)
这是你写代码的主战场。它比记事本强大在哪？
*   **Syntax Highlighting (高亮)**：代码是彩色的。变量、函数、字符串一目了然。
*   **IntelliSense (智能感知)**：输入 `pr`，它会自动提示 `print`。这不仅省键盘，更能防止拼写错误。
*   **Linting (实时纠错)**：少写了一个括号？它会立刻画上**红色波浪线**。

### 2.3 底部：集成终端 (Integrated Terminal)
按快捷键 `Ctrl + \`` (反引号) 或 `Ctrl + J` 呼出。
*   **机制**：这也是一个标准的 Shell（Zsh 或 PowerShell）。
*   **联动**：当你打开 IDE 时，这个终端通常会自动定位到你当前打开的文件夹下（但这不绝对，下一节我们会验证）。

---

## 3. 核心机制：全能指挥官 (Command Palette)

IDE 里有成百上千个功能，不要去菜单栏里翻。
学会召唤 **Command Palette (命令面板)**，这是专业用户的标志。

*   **Mac**: `Cmd + Shift + P`
*   **Windows**: `Ctrl + Shift + P`

这是一个万能搜索框。输入你想要的操作（中英文皆可），它就会执行。
*   想选解释器？搜 `Select Interpreter`。
*   想换颜色？搜 `Color Theme`。
*   想格式化代码？搜 `Format Document`。

---

## 4. 实战：复刻 M1L5 的工作流

让我们在 IDE 里把 M1L5 的脚本再跑一次，体验效率的飞跃。

### 4.1 打开文件夹 (Open Folder)
**这是极其重要的一步**。不要只打开文件，要打开整个文件夹。
`File -> Open Folder` -> 选择你的 `python_scripts` 文件夹。

### 4.2 配置解释器 (Interpreter)
IDE 需要知道用哪个 Python（是系统的？还是我们配置的 `ai_course`？）来运行代码。
1.  呼出 **命令面板**。
2.  输入 `Python: Select Interpreter`。
3.  **关键选择**：在列表中找到带有 `ai_course` 字样的选项。
    *   *观察状态栏*：右下角应该显示 `3.12.x ('ai_course')`。

### 4.3 编写与运行
1.  新建文件 `ide_demo.py`。
2.  输入代码（体验自动补全）：
    ```python
    import sys
    print("当前解释器路径:")
    print(sys.executable)
    ```
3.  **执行的三种方式**：
    *   **方式 A (小白)**：点击右上角的绿色“播放”按钮。
    *   **方式 B (老手)**：在底部终端手打 `python ide_demo.py`。
    *   **透视真相**：当你点击播放按钮时，请观察底部终端。IDE 其实是自动帮你输入了一行命令。它并没有魔法，它只是帮你省了打字的时间。

## 5. 工程自检 (Engineering Checkpoint)

本课重在验证 IDE 配置的正确性。

### ✅ 验收标准 (Pass Criteria)
1.  **解释器验证**：运行 `ide_demo.py`，输出的路径必须包含 `envs/ai_course`。如果显示的是 `/usr/bin/python`，说明配置错误。
2.  **终端联动**：打开底部终端，输入 `pwd`，路径应指向你当前打开的文件夹。
3.  **命令面板**：能熟练呼出面板并搜索出 `Theme` 选项。

### 🛑 常见故障 (Fail & Triage)
*   **故障 1**：代码全是白色，没有颜色高亮。
    *   *诊断*：你可能把文件名存成了 `.txt`，或者右下角语言模式没选 Python。
*   **故障 2**：`ImportError` 但在终端里能跑。
    *   *诊断*：IDE 选择的解释器（右下角）和你终端里激活的环境不一致。请重新 Select Interpreter。
*   **故障 3**：点击播放按钮后报错 "File not found"。
    *   *诊断*：文件名里可能有空格？或者 IDE 的终端启动位置不对。尝试自己在终端里 `cd` 到正确位置运行。
