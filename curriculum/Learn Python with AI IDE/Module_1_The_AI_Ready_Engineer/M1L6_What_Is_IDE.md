## 1. Philosophy: 工具主权与 AI IDE 生态

### 1.1 主流 IDE 与 AI 增强件
在正式开始前，我们需要了解目前工程界主流的开发工具。虽然我们使用的是 Antigravity，但这些知识在其他工具中是通用的：

*   **VS Code**：当代软件开发的“中央车站”。几乎所有主流 IDE（包括 Antigravity）都是基于它的开源框架构建的。
*   **AI 原生 IDE**：如 **Cursor**、**Kirokiro** 以及字节跳动出品的 **Trae**。
*   **AI 增强插件**：即使是普通的 IDE，也可以通过安装插件获得 AI 辅助能力。例如 **GitHub Copilot** (基于 OpenAI 模型的代码助手)、**Claude Code** 或 **Codex**。

### 1.2 为什么需要“工具主权” (Tool Sovereignty)？
在 AI 时代，学习 IDE 的重点不是点击按钮，而是掌握**协作的第一现场**。

*   **IDE 是互动的“唯一真值”**：虽然你可以通过网页、终端与 AI 互动，但代码最终的落地、调试与运行，最主流且最稳健的场景依然是在 IDE 中。
*   **看清 AI 的每一个动作**：AI 修改了哪个文件？在终端执行了什么指令？环境是否被污染？如果不掌握 IDE，你对 AI 的协作就像是“隔山打牛”。只有拿回工具主权，你才能在 AI 出错时，依然拥有对工程的绝对控制力。

## 2. The Raw Logic：界面感知与“生存配置”

### 2.1 界面初始化 (The Initial Setup)
打开 Antigravity 后，我们需要将界面调整为最经典的“工程模式”。请确保以下三个板块清晰可见：

1.  **左侧栏 (Primary Side Bar)**：点击左上角的方块图标。这是管理工程资产的区域。
2.  **隐藏 AI 面板 (Toggle Agent)**：暂时将其**关闭**。在学习阶段，我们需要先看清工具的本原。
3.  **底部面板 (Panel)**：
    *   **Mac**: `Command + J`
    *   **Windows**: `Ctrl + J`
    这是你之前熟悉的终端所在的区域。

现在，你应该看到一个稳定的布局：**左侧资源管理器**、**右上编辑区**、**下部控制区**。

### 2.2 核心图标与扩展 (Extensions)
在左侧边栏，你会看到几个核心功能图标。作为 Python 工程师，最常用的是：
*   **资源管理器 (Explorer)**：管理当前项目的文件夹和文件。
*   **扩展 (Extensions)**：安装增强功能的“插件商店”。

### 2.3 宿主与插件的关系
IDE 就像一个精密的硬件支架（宿主），而**扩展 (Extensions)** 则是为它注入灵魂的功能模块。
1.  **Python 扩展**：这是必备组件。它赋予了 IDE 识别、运行、调试和智能提示 Python 代码的能力。
2.  **中文语言包 (Simplified)**：安装后重启，界面将切换为中文简体。

## 3. Instruction：关联环境与项目

### 3.1 搬入项目文件夹
将我们在 M1L5 中辛苦打磨的“第一批移民”搬进 IDE。这也是我们在 L5 手动创建的那三个 `.py` 脚本文件（`hello.py` 等）。
*   点击菜单栏 `文件 (File)` -> `打开文件夹 (Open Folder)`。
*   选择该文件夹。此时，你可以在左侧看到所有的文件。

### 3.2 配置解释器：插座与插头 (Recall M1L4)
> **🔌 隐喻回调**
> 环境 `ai_course` 是我们的**专用电源插座**。现在，我们要为 IDE 这台设备接通电源。

1.  呼出**命令面板** (`Cmd/Ctrl + Shift + P`)。
2.  搜索并选择 `Python: Select Interpreter` (选择解释器)。
3.  在列表中选中带有 `ai_course` 标识的一项。
    *   *观察*：IDE 右下角状态栏应显示 `ai_course`。

## 4. Practice：复刻与验证

### 4.1 创建 IDE 演示脚本
1.  **创建文件**：在左侧**资源管理器**中右键点击文件夹空白处，选择 `新建文件 (New File)`，命名为 `ide_demo.py`。
2.  **进入编辑状态**：单击刚创建的 `ide_demo.py`，你会在右上的**编辑区**看到一个空白的输入窗口。
3.  **编写代码**：就像在记事本中输入文字一样，在窗口中输入：
    ```python
    print("Hello AI IDE")
    ```
    *你会发现：输入 `pri` 时会出现提示补全。这其实是 IDE 安装了 Python 插件后的“词典映射”功能，它能认出语法。未来我们加装 AI 引擎后，它将不再只是认词，而是能够理解你的意图。*

### 4.2 运行对比：手打 vs 一键
*   **方法 A (传统终端)**：在 IDE 底部的**终端**里，手动输入 `python ide_demo.py`。
*   **方法 B (IDE 一键运行)**：点击编辑器右上角的**运行按钮**（三角形图标）。

**硬核观察**：点击运行按钮后，请死死盯着底部终端。你会发现 IDE 帮你输入了一行极长的指令。
**结论**：IDE 并不是魔法，它是一个**集成的自动化平台**。除了运行代码，未来你还将在在这里管理数据库、管理 Git 仓库。它是你作为工程师的**唯一完整工作站**。
