# Module 1 Lesson 6: 工程结构与工作区 (Workspaces)

> **Module 1: The AI-Ready Engineer**
> **目标**：建立专业的文件组织意识。理解“文件夹”即是整个项目的边界。

---

## 1. 理论：打开“文件” vs 打开“工程”

在新手时期，你可能习惯于“双击一个文件”去编辑。但在现代软件开发中，我们不再孤立地看一个文件，而是以 **工程/工作区 (Project/Workspace)** 为单位进行管理。

**为什么必须“打开文件夹”？**
1.  **全局视野**：IDE 能看到你整个项目里的所有文件，从而提供跨文件的代码提示。
2.  **上下文关联**：AI 助手（如 Antigravity）需要根据整个文件夹的内容来理解你的项目意图。
3.  **路径基准**：你在代码中引用的相对路径，都是相对于你“打开的那个根目录”而言的。

---

## 2. 机制：工作区根目录 (Root Directory)

当你把文件夹拖入 Antigravity 时，这个文件夹就成为了 **Root (根目录)**。

想象一下，如果你的项目是一座大楼：
*   **Root** 就是大楼的地基。
*   **子文件夹** 是不同的楼层（如 `scripts/`, `data/`, `docs/`）。
*   **文件** 就是每个房间里的具体物品。

**工程规范建议**：
*   **所有源码** 保持在同一个根目录下。
*   **分类明确**：不要把数据、文档和脚本混在一起。

---

## 3. 指令：在 IDE 中管理文件

### 3.1 创建工程目录
我们在终端里做一个硬核操作：
```bash
cd ~
mkdir -p projects/ai_course_lab
```

### 3.2 在 Antigravity 中打开
1.  选择菜单：`File` -> `Open Folder...`。
2.  定位到刚才创建的 `ai_course_lab`。
3.  点击打开。现在，你的 IDE 侧边栏 (Explorer) 应该显示了这个文件夹。

### 3.3 尝试新建结构
点击侧边栏的图标，分别创建一个文件夹和一个文件：
1.  新建文件夹：`module1_lab/`
2.  在文件夹内新建文件：`app.py`

---

## 4. 硬核练习

1.  **路径验证**：在 `app.py` 中写下：
    ```python
    import os
    print("当前工作目录是:", os.getcwd())
    ```
2.  **运行并观察**：运行后看输出。你会发现输出的路径正是你刚才在 IDE 中打开的那个 **工程根目录**。
3.  **跨级别练习**：尝试在根目录下创建一个 `notes.txt`，看看在 `app.py` 中运行代码时，资源管理器里是否能同时清晰地看到刚才创建的所有层级。

---

> [!TIP]
> **本阶段总结**：
> 一个整洁的工程结构是工程成熟度的标志。永远记住：**先打开文件夹，再开始写代码。** 这不仅是你的习惯，也是让 AI 助手变聪明的关键策略。
