# Module 1 Lesson 8: 智能助手 —— AI Sidebar Agent (Expanded)

> **Module 1: The AI-Ready Engineer**
> **目标**：认识你的超级副官。理解 AI 如何跨越“意图”与“代码”之间的鸿沟。

---

## 1. 理论：从“搜索引擎”到“智能体 (Agent)”

如果你还觉得 AI 只是一个像网页版 ChatGPT 那样的聊天框，那你就落后了。

在 **Antigravity** 中，你面对的是一个 **Agent (智能体)**。
*   **传统搜索**：你得先遇到问题，复制报错，去浏览器搜索，再复制粘贴回来。
*   **侧边栏 Agent**：它驻留在你的编辑器内部。它不需要你复制粘贴，它已经在阅读你的代码了。

**Agent 的核心特质：**
1.  **感知力 (Perception)**：它能感知到你的光标位置、你刚才写的注释，甚至你还没保存的文件。
2.  **行动力 (Action)**：在获得你授权的前提下，它可以直接写代码、改文件、跑测试。
3.  **理解力 (Reasoning)**：它能理解你模糊的需求（如“我想让这段代码跑得快一点”），并将其转化为精确的工程步骤。

---

## 2. 机制：Agent 是如何调动工具的？

一个 Agent 之所以强大，不是因为它“背熟了语法”，而是因为它会用 **Tools (工具)**。

在 Antigravity 的底层，Agent 拥有一套“工具箱”：
*   **File Tool**：读取、搜索和修改工作区内的任何文件。
*   **Terminal Tool**：在你的 `(ai_course)` 环境中执行 Shell 命令。
*   **Debugger Tool**：在程序崩溃时，监控变量的变化。

> [!IMPORTANT]
> **接轨 Stanford CS146S**：
> 学习如何调动工具是现代软件开发的必修课。这种“模型+工具”的模式正是斯坦福后续课程中 **AI Agent 系统设计** 的核心逻辑。

---

## 3. 指令：与 Agent 共创

请确保在你的 `ai_lab_v1` 工作区内。打开侧边栏 (Chat)。

### 3.1 深度上下文测试
不要粘贴任何代码，直接问它：
`“根据我刚才在 main.py 里写的路径逻辑，如果我把 assets/config.info 删了，会发生什么？”`
**观察**：它是否能立刻定位到你的代码行，并给出准确的逻辑预判？

### 3.2 意图驱动开发 (Intent-Driven)
输入：`“帮我写一个 python 脚本，用于扫描当前目录下所有文件的行数，并在终端里打印总结报告。”`
**观察 Agent 的执行步调：**
1.  它会向你解释它的思路（使用了哪些 Python 库）。
2.  它会展示代码片段。
3.  它会尝试在你的工作区创建一个新文件。
4.  它会问你：“我是否可以执行这个操作？”

---

## 4. 重点：AI 并不是“自动驾驶”

虽然 Agent 很强，但它目前的阶段更像是**“高级辅助驾驶” (L2/L3)**。
*   **它是概率性的**：它给出的代码取决于它的训练数据。
*   **它可能有幻觉**：它可能引用一个根本不存在的 Python 库或文件。

所以，在接下来的 **Module 2: Python Essentials with AI IDE** 中，你的主要任务是学习如何**“审计”**它的每一行产出。

---

## 5. 硬核练习

1.  **环境联动挑战**：
    告诉 Agent：`“帮我检查一下我当前的 Python 运行环境是否正确配置为了 ai_course。”`
    看看它是否会通过运行终端命令 `conda info --envs` 来进行自我验证。

2.  **Bug 模拟共创**：
    在 `main.py` 故意写一个错得离谱的逻辑（比如 `1 + "1"`）。运行后，在侧边栏点击“Fix with AI”。观察它是否能同时看到你的代码、你的报错信息，并给出不仅正确而且带有原理解释的修复建议。

3.  **长距离搜索**：
    让 Agent 在整个工作区里寻找所有包含 “logic” 字样的代码行。体验它比传统 `grep` (M1_3 学过) 更强大的自然语言语义识别能力。
