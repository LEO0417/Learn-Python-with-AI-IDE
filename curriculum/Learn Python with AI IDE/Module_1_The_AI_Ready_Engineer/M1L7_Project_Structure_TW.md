## 1. 工程邏輯：目錄是代碼的「容器」

一個完整的**項目 (Project)** 除了代碼，還可能包含數據、文檔、配置以及依賴說明。

### 1.1 從單檔案到目錄樹
如果把代碼比作單一的實物，那麼**項目目錄**就是存放他們的「貨架」。
*   **混亂佈局**：所有檔案（代碼、日誌、臨時檔案）全部堆在一起，難以維護且容易誤刪。
*   **規範佈局**：優秀的開發者會透過子目錄管理，將代碼、數據與文檔隔離。

### 1.2 為什麼必須規範？
1.  **降低維護成本**：當你打開側邊欄時，清晰的目錄結構能直觀地展示工程的全貌。
2.  **提升協作效率**：在下一課中我們要引入 **AI 輔助能力**。結構清晰的項目能讓 AI 透過檔案名（如 `scripts` 或 `data`）迅速定位到相關背景，減少溝通偏差。

---

## 2. 理解映射：IDE 檔案總管與物理磁碟

IDE 左側的**檔案總管 (Explorer)** 並不是一個獨立的虛擬空間。它的物理本質是**本地磁碟資料夾在 IDE 中的「即時鏡像」**。

### 2.1 即時同步邏輯
你可以進行以下驗證：
1.  在 IDE 的左側空白處右鍵點擊「新建資料夾」，命名為 `my_folder`。
2.  同時打開你的電腦系統窗口（Mac 的 Finder 或 Windows 的檔案總管），找到對應的物理路徑。
3.  你會發現，資料夾在 IDE 裡出現的瞬間，本地硬碟上也同步生成了。

**核心準則**：你在檔案總管裡的每一次點擊、創建或刪除，都是對本地物理硬碟的真實操作。IDE 只是為你提供了一個更高效的**可視化管理界面**。

## 3. 實作指引：檔案總管的高效操作

### 3.1 核心操作集
在檔案總管側邊欄上，以下是開發者最常用的右鍵動作：
*   **新建檔案 (New File)**：創建新的腳本或文本文檔。
*   **新建資料夾 (New Folder)**：建立邏輯分類。
*   **在檔案總管中顯示 (Reveal in Finder/Explorer)**：快速跳轉到本地系統窗口。

### 3.2 項目名片：README.md
在軟件工程中，`README.md` 是一個標準化的說明文檔。
*   **用途**：記錄項目功能、學習筆記以及執行方法。
*   **優勢**：IDE 提供了對 Markdown 的原生支持，不僅可以編輯，還支持即時渲染預覽。

## 4. 重點實驗：重構項目工程

我們將把 L5 和 L6 的散裝代碼進行一次標準化的「搬遷」。

### 4.1 構建標準目錄
在 IDE 的檔案總管中執行以下操作：
1.  **創建資料夾**：在根目錄下創建一個資料夾 `scripts`。
2.  **移動資產**：用鼠標選中之前創建的 `ide_demo.py` 和 L5 的腳本檔案，直接**拖拽**進 `scripts` 資料夾中。
    *   *說明*：當 IDE 詢問「是否移動」時，點選確定。這會同步修改本地物理路徑。
3.  **創建數據區**：創建一個名為 `data` 的空資料夾，用於後續課程存放外部數據。

### 4.2 編寫項目說明
1.  在根目錄下新建檔案 `README.md`。
2.  輸入一段標準化的項目描述：
    ```markdown
    # 我的第一個 Python 工程
    這是在 Module 1 創建的標準練習項目。
    
    - `scripts/`: 存放 Python 邏輯腳本。
    - `data/`: 存放相關數據檔案。
    ```
3.  **預覽驗證**：點擊編輯器右上角的「預覽」按鈕，查看排版效果。
