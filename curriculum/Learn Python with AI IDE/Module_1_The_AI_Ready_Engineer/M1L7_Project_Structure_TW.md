## 1. Philosophy: 項目是代碼的「容器」

在 Module 1 的練習中，你可能已經創建了好幾個 `.py` 檔案。但在真正的軟體開發中，一個完整的**項目 (Project)** 除了代碼，還可能包含資料、文件和配置。

### 1.1 從單檔案到資料夾
如果把代碼比作「家具」，那麼**項目資料夾**就是你的「房間」。
*   如果你把所有家具都堆在門口，你很難找到想要的東西。
*   優秀的工程師會在房間裡擺放「櫃子」（子資料夾），將代碼和資料分類存放。

### 1.2 為什麼必須整潔？
1.  **降低認知負擔**：當你開啟側邊欄，清晰的目錄結構會告訴你「這裡是代碼，那是資料」。
2.  **AI 友好的索引**：下一課我們要用的 **AI Agent** 其實是個「讀圖員」。如果你把項目整理得井井有條，AI 掃一眼資料夾名字（如 `scripts` 或 `docs`），就能立刻定位到它該去哪裡輔助你。

## 2. The Raw Logic：所見即所得的同步

IDE 左側的**檔案總管 (Explorer)** 並不是一個獨立的虛擬空間。它的物理本質是**本地磁碟資料夾在 IDE 中的「實時鏡像」**。

### 2.1 兩個宇宙的同步
你可以做一個有趣的實驗：
1.  在 IDE 的左側空白處右鍵點擊「新建資料夾」，命名為 `my_folder`。
2.  同時開啟你的電腦視窗（Mac 的 Finder 或 Windows 的檔案資源管理器），找到對應的物理路徑。
3.  你會發現，資料夾在 IDE 裡出現的瞬間，本地硬碟上也同步出現了。

**邏輯核心**：你在檔案總管裡的每一次點擊、拖拽和重命名，都是在直接操作你電腦裡的硬碟。IDE 只是為你提供了一個更高效的**操作界面**。

---

## 3. Instruction：檔案總管的生存技巧

### 3.1 核心右鍵選單
在檔案總管（側邊欄）上點擊右鍵，你會看到幾個最重要的工程動作：
*   **新建檔案 (New File)**：不僅是 `.py`，以後我們還會創建文本、配置等。
*   **新建資料夾 (New Folder)**：建立分類。
*   **在資源管理器中顯示 (Reveal in Finder/Explorer)**：如果你迷路了，這個功能會直接幫你開啟本地電腦視窗。

### 3.2 檔案的「名片」：README.md
在所有的規範項目中，都會有一個特殊檔案：`README.md`。
*   **用途**：它是給人類閱讀的「項目說明書」。
*   **優勢**：IDE 對這種檔案提供了完美的預覽支持。你可以用它記錄學習心得、執行方法。

## 4. Practice：整理你的「數位工作室」

讓我們把 L5 和 L6 的散裝練習進行一次正式的搬家。

### 4.1 手動構建工程藍圖
在 IDE 的檔案總管中，完成以下動作：
1.  **建櫃子**：右鍵點擊工程根目錄，創建一個資料夾名為 `scripts`。
2.  **搬家**：用滑鼠選中你之前創建的 `ide_test.py` （或 `ide_demo.py`）和 L5 的那些小腳本，直接**拖拽**進 `scripts` 資料夾中。
    *   *觀察*：IDE 可能會彈出視窗詢問「是否要移動檔案？」，大膽點擊「確定」。
3.  **建資料區**：創建一個名為 `data` 的空資料夾，準備未來存放資料。

### 4.2 撰寫第一份項目說明
1.  在根目錄下新建檔案 `README.md`。
2.  輸入一段文字。例如：
    ```markdown
    # 我的第一個 Python 項目
    這是我在 Module 1 建立的學習區。包含以下內容：
    - `scripts/`: 學習 Python 基礎操作的腳本。
    - `data/`: 未來存放練習資料的區域。
    ```
3.  **預覽演示**：點擊編輯器右上角的「預覽」圖標（通常是一個帶放大鏡的文件圖標），感受 Markdown 檔案帶來的專業顯示效果。

## 5. Engineering Checkpoint (工程自檢)

### ✅ 驗收標準 (Pass Criteria)
1.  **目錄整潔**：根目錄下只有 `scripts`、`data` 資料夾和 `README.md`，不再有散亂的 `.py` 檔案。
2.  **同步驗證**：在 IDE 裡重命名一個資料夾，確認本地電腦對應的資料夾也跟著變了。
3.  **Markdown 預覽**：能正常查看 `README.md` 的渲染效果。
