## 2. The Raw Logic：IDE 的「三位一體」架構

我們使用的 IDE 名為 **Antigravity**。它的物理本質並不神秘，它的核心是将三個核心生產力工具「縫合」到了同一個界面下：

1.  **File Explorer (資源管理器)**：你的文件地圖。取代了 `ls` 和 `cd`。
2.  **Code Editor (智能編輯器)**：你的寫作台。它比記事本多出的功能叫做 **IntelliSense (智能感知)** 和 **Linting (實時糾錯)**。
3.  **Integrated Terminal (集成終端)**：你的控制台。它就是你之前一直在用的 Shell。

**關鍵認知**：即使在 IDE 裡，底層的物理規則依然沒變。點擊「運行」按鈕的本質，依然是 IDE 悄悄在底部的終端裡幫你打了一行 `python xxx.py`。**IDE 只是幫你省掉了那些機械性的重複勞動，而不是替代了原理。**

## 3. Instruction：核心機制與配置

### 3.1 萬能入口：Command Palette (命令面板)

在專業工程師的世界裡，鼠標是效率的敵人。掌握**命令面板**是區分專業開發者與業餘愛好者的標誌。

*   **喚醒指令**：
    *   **Mac**: `Cmd + Shift + P`
    *   **Windows**: `Ctrl + Shift + P`
*   **邏輯**：這是一個「全量搜索框」。無論是切換主題、查找文件，還是配置 Python，你只需要在這裡輸入關鍵詞，它就會執行相應的操作。

### 3.2 配置解釋器：插座與插頭 (Recall M1L4)

這是本課最重要的配置步驟。IDE 必須知道應該用哪個 Python 來執行你的代碼。

> **🔌 插座與插頭隱喻**
> 在 M1L4 中，我們創建了虛擬環境 `ai_course`。你可以把它想象成一個牆上的**專用電源插座**，裡面提供了特定的電壓和工具。
> 現在的 IDE 就像一台昂貴的精密機器。點擊運行之前，你必須選擇正確的**插頭 (Interpreter)**。

1.  喚出**命令面板**。
2.  搜索並選擇：`Python: Select Interpreter`。
3.  在出現的列表中，**必須**選擇帶有 `ai_course` 標識的一項。
    *   *觀察*：IDE 界面右下角的狀態欄此時應顯示 `3.13.x ('ai_course')`。如果不是，說明你的設備還沒通電。

## 4. Practice：複刻與驗證

我們將通過在 IDE 中重跑 M1L5 的邏輯，來驗證整個工程鏈路的通暢。

### 4.1 打開文件夾 (Open Folder)
**工程準則**：不要僅打開單個代碼文件，一定要打開該文件所屬的**根文件夾**。這會讓 IDE 建立起整個項目的索引。
*   `File -> Open Folder` -> 選擇你在 M1L2 中創建的 `my_ai_project`。

### 4.2 創建演示腳本
1.  在左側文件瀏覽器中右鍵選擇 `New File`，命名為 `ide_test.py`。
2.  寫入以下代碼（體驗**智能補全**，當你輸入 `pri` 時，IDE 應該會提示 `print`）：
    ```python
    import sys
    from pathlib import Path

    print("--- 解釋器鏈接驗證 ---")
    # 打印當前正在運行代碼的 Python 解釋器路徑
    print(f"當前使用的插座地址: {sys.executable}")
    
    # 回調 L2 的路徑概念
    print(f"當前文件的物理錨點: {Path(__file__).resolve()}")
    ```

### 4.3 運行並透視真相
點擊 IDE 右上角的綠色三角形（播放按鈕）。

**硬核觀察**：觀察底部的 **Terminal** 窗口。你會發現它自動出現了一長串命令。那就是 IDE 幫你自動輸入並執行的終端指令。

## 5. Engineering Checkpoint (工程自檢)

### ✅ 驗收標準 (Pass Criteria)
1.  **路徑驗證**：`ide_test.py` 輸出的解釋器路徑必須包含 `envs/ai_course` (Mac/Linux) 或 `envs\ai_course` (Windows)。
2.  **狀態欄驗證**：右下角的 Python 環境提示準確顯示了 `ai_course`。
3.  **聯動驗證**：在 IDE 內部的終端輸入 `pwd`，顯示的路徑應正好是你打開的文件夾路徑。

### 🛑 常見故障處理
*   **沒高亮（全白）**：確認文件名後綴是 `.py` 而不是 `.txt`。
*   **ImportError**：通常是因為「插頭」拔錯了，請重新執行 `Select Interpreter`。
