## 1. 主流 IDE 與 AI 生態
在正式開始前，我們需要再次明確 IDE 的定義，並了解目前工程界主流的開發工具：

*   **IDE (Integrated Development Environment)**：整合開發環境。它不僅僅是一個寫代碼的文本編輯器，而是一個集成了檔案管理、代碼編輯、終端機控制和調試功能的「軍火庫」。
*   **AI IDE**：在傳統 IDE 的基礎上，深度集成了 AI 能力的開發工具。
    *   **Antigravity**：我們課程使用的教學版 IDE。
    *   **Cursor**：業界首款流行的 AI 原生 IDE。
    *   **Windsurf**：由 Codeium 出品，其 Flow 功能提供了規範的 AI 協作編程方式。
    *   **Kilo**：新興的 AI 原生 IDE，專注於極致性能與 AI 深度融合。
    *   **Trae**：字節跳動出品的 AI IDE。
*   **AI 增強外掛**：即使是傳統的 IDE (如 VS Code)，也可以透過安裝外掛獲得 AI 輔助能力。這些外掛包括：
    *   **Gemini CLI Companion**
    *   **Gemini Code Assist**
    *   **Claude Code for VS Code**
    *   **Codex – OpenAI’s coding agent**

## 2. The Raw Logic：界面感知與「生存配置」

### 2.1 界面初始化 (The Initial Setup)
打開 Antigravity 後，我們需要將界面調整為最經典的「工程模式」。請熟悉以下三個核心區域及其快捷鍵區分：

1.  **檔案總管 (Explorer) - 左側欄**
    *   **功能**：管理工程資產，瀏覽檔案結構。
    *   **快捷鍵**：
        *   **Mac**: `Cmd + Shift + E`
        *   **Windows**: `Ctrl + Shift + E`

2.  **編輯器 (Editor) - 右上核心區**
    *   **功能**：編寫代碼的主戰場。
    *   **快捷鍵**：`Cmd/Ctrl + 1` (快速切回第一個編輯視窗)

3.  **終端機面板 (Panel/Terminal) - 底部控制區**
    *   **功能**：執行指令，查看輸出。
    *   **快捷鍵**：
        *   **Mac**: `Cmd + J` 或 `Ctrl + ~`
        *   **Windows**: `Ctrl + J` 或 `Ctrl + ~`

此外，左側通常還有一個 **AI 側邊欄 (Sidebar Agent)**，暫時將其**隱藏**，我們需要專注於核心工程界面。

現在，你應該看到一個穩定的佈局：**左側檔案總管**、**右上編輯區**、**下部終端機區**。

### 2.2 核心邏輯：宿主與延伸模組 (Host & Extensions)

IDE 就像一個精密的硬體支架（宿主），但這僅僅是軀殼。**延伸模組 (Extensions)** 才是為它注入靈魂的功能模組。

在左側邊欄，點擊那個「四個方塊」組成的圖標，這就是**外掛商店**。作為 Python 工程師，我們必須手動安裝兩個「靈魂」：

1.  **Python 延伸模組**
    *   **作用**：賦予 IDE 識別 Python 語言的能力。沒有它，IDE 就只會被當成普通的記事本。它負責代碼高亮、智能提示、以及代碼執行。
2.  **中文語言包 (Traditional Chinese)**
    *   **作用**：將界面語言從英文切換為繁體中文。安裝後通常需要重啟 IDE 生效。

*除此之外，檔案總管 (Explorer) 是我們最常駐的左側面板，用於管理所有檔案。*

## 3. Instruction：關聯環境與項目

### 3.1 搬入項目資料夾
將我們在 M1L5 中辛苦打磨的「第一批移民」搬進 IDE。這也是我們在 L5 手動創建的那三個 `.py` 腳本檔案（`hello.py` 等）。
*   點擊菜單欄 `文件 (File)` -> `開啟資料夾 (Open Folder)`。
*   選擇該資料夾。此時，你可以在左側看到所有的檔案。

### 3.2 配置解釋器：插座與插頭 (Recall M1L4)
> **🔌 隱喻回調**
> 環境 `ai_course` 是我們的**專用電源插座**。現在，我們要為 IDE 這台設備接通電源。

我們有兩種方式來完成這次「通電」：

**方式 A：透過指令面板 (標準做法)**
1.  呼出**指令面板** (`Cmd/Ctrl + Shift + P`)。
2.  搜尋並選擇 `Python: Select Interpreter` (選擇解釋器)。
3.  在列表中選中帶有 `ai_course` 標識的那一項。

**方式 B：透過狀態欄 (快捷做法)**
1.  打開任意一個 `.py` 檔案（例如 `hello.py`）。
2.  看向 IDE 的**右下角狀態欄**，你會看到顯示當前 Python 版本的地方（或者顯示 `Select Interpreter`）。
3.  直接點擊它，這會彈出和方式 A 一樣的選擇列表。
4.  選擇 `ai_course`。

*操作完成後，請確認右下角已顯示 `ai_course` 字樣。*

## 4. Practice：複刻與驗證

### 4.1 創建 IDE 演示腳本
1.  **創建檔案**：在左側**檔案總管**中右鍵點擊資料夾空白處，選擇 `新建檔案 (New File)`，命名為 `ide_demo.py`。
2.  **進入編輯狀態**：單擊剛創建的 `ide_demo.py`，你會在右上的**編輯區**看到一個空白的輸入視窗。
3.  **編寫代碼**：就像在記事本中輸入文字一樣，在視窗中輸入：
    ```python
    print("Hello AI IDE")
    ```
    *你會發現：輸入 `pri` 時會出現提示補全。這其實是 IDE 安裝了 Python 模組後的「詞典映射」功能，它能認出語法。未來我們加裝 AI 引擎後，它將不再只是認詞，而是能夠理解你的意圖。*

### 4.2 運行對比：手動 vs 一鍵
*   **方法 A (傳統終端機)**：在 IDE 底部的**終端機**裡，手動輸入 `python ide_demo.py`。
*   **方法 B (IDE 一鍵執行)**：點擊編輯器右上角的**執行按鈕**（三角形圖標）。

**硬核觀察**：點擊執行按鈕後，請死死盯著底部終端機。你會發現 IDE 幫你輸入了一行極長的指令。
**結論**：IDE 並不是魔法，它是一個**集成的自動化平臺**。除了運行代碼，未來你還將在這裡管理資料庫、管理 Git 倉庫。它是你作為工程師的**唯一完整工作站**。
