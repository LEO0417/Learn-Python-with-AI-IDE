## 1. 主流 IDE 與 AI 生態
在正式開始前，我們需要再次明確 IDE 的定義，並了解目前工程界主流的開發工具：

*   **IDE (Integrated Development Environment)**：整合開發環境。它不僅僅是一個寫代碼的文本編輯器，而是一個整合了檔案管理、代碼編輯、終端機控制和調試功能的「軍火庫」。
*   **AI IDE**：在傳統 IDE 的基礎上，深度整合了 AI 能力的開發工具。
    *   **Antigravity**：我們課程使用的教學版 IDE。
    *   **Cursor**：業界首款流行的 AI 原生 IDE。
    *   **Windsurf**：由 Codeium 出品，其 Flow 功能提供了規範的 AI 協作編程方式。
    *   **Kilo**：新興的 AI 原生 IDE，專注於極致性能與 AI 深度融合。

    *   **Trae**：字節跳動出品的 AI IDE。
*   **AI 增強外掛**：即使是傳統的 IDE (如 VS Code)，也可以透過安裝外掛獲得 AI 輔助能力。這些外掛包括：
    *   **Gemini CLI Companion**
    *   **Gemini Code Assist**
    *   **Claude Code for VS Code**
    *   **Codex – OpenAI’s coding agent**

## 2. 底層認知：界面佈局與邏輯結構

### 2.1 界面佈局 (The Interface Layout)
打開 Antigravity 後，我們需要將其調整為標準的工程開發佈局。請熟悉以下三個核心功能區域：

1.  **檔案總管 (Explorer) - 左側面板**
    *   **功能**：管理工程檔案，瀏覽目錄結構。
    *   **快捷鍵**：
        *   **Mac**: `Cmd + B`
        *   **Windows**: `Ctrl + B`

2.  **編輯器 (Editor) - 右上核心區**
    *   **功能**：編寫代碼、閱讀邏輯的主要區域。
    *   **Pro Tip (視力友好)**：使用 `Cmd/Ctrl + +` 或 `-` 可以快速縮放整個界面的文字大小，找到最適合你的閱讀比例。

3.  **終端機面板 (Panel/Terminal) - 底部區域**
    *   **功能**：輸入指令，查看解釋器輸出。
    *   **快捷鍵**：
        *   **Mac**: `Cmd + J`
        *   **Windows**: `Ctrl + J`

### 2.2 核心邏輯：主體與延伸模組 (Core & Extensions)

IDE 作為一個高度可定制的軟件，其功能很大程度上依賴於它的**延伸模組 (Extensions)** 系統。

在左側邊欄點擊「延伸模組」圖標（四個方塊）進入商店。對於 Python 開發，必須確保兩個核心組件的安裝：

1.  **Python 延伸模組**
    *   **作用**：賦予 IDE 識別、高亮、補齊 Python 代碼的能力，並負責關聯後台解釋器。
2.  **中文語言包 (Traditional Chinese)**
    *   **作用**：將界面語言本地化。

### 2.3 核心生存配置：告別低級錯誤

在正式編寫代碼前，我們需要調整兩個能夠大幅提升幸福感的「生存開關」：

1.  **開啟自動儲存 (Auto Save)**
    *   **操作步驟**：
        1.  呼出指令面板 (`Cmd/Ctrl + Shift + P`)。
        2.  輸入並選擇：`Preferences: Open Settings (UI)`。
        3.  在右側搜尋框輸入：`files autosave`。
        4.  找到 **Files: Auto Save** 項，在下拉框中選擇 **onFocusChange**。
    *   **優勢**：當你點擊「執行」按鈕或切換到終端機時，IDE 會自動儲存。這既能保證執行的是最新代碼，又不會因頻繁儲存而讓專案的本地歷史記錄 (Timeline) 變得過於瑣碎。
2.  **識別紅色波浪線 (The Red Warning)**
    *   **認知**：當你輸入錯誤的語彙（如 `printt`）時，IDE 會立即在下方標註紅色波浪線。
    *   **準则**：**不要帶著波浪線執行代碼**。這是 IDE 提前幫你識別出的「執行必崩」點。

> **說明**：左側通常還提供 **AI 側邊欄 (Sidebar Agent)**。在本模組初期掌握基礎工程能力時，建議先將其收起，專注於核心界面。

### 3.2 配置解釋器：環境映射 (Recall M1L4)

> **🔌 邏輯回調**
> 在 L4 創建的虛擬環境 `ai_course` 是代碼運行的動力源。現在，我們需要將 IDE 映射到這個特定的環境。

**方式 A：透過指令面板 —— IDE 的「傳送門」 (推薦)**
作為專業開發者，你應該習慣透過快捷鍵呼出指令面板 (`Cmd/Ctrl + Shift + P`)。它像是一個直達任意配置的傳送門，你不需要記住複雜的選單路徑，只需輸入關鍵詞即可：
1.  呼出指令面板。
2.  輸入並選擇 `Python: Select Interpreter`。
3.  在列表中選中帶有 `ai_course` 標識的版本。

**方式 B：透過狀態欄**
1.  打開任意 `.py` 檔案。
2.  點擊 IDE **右下角狀態欄**的 Python 版本顯示區。
3.  在彈出的列表中選擇 `ai_course`。

*配置完成後，右下角應明確顯示 `ai_course` 字樣。*

## 4. 重點實驗：從手動執行到整合執行

### 4.1 創建驗證腳本
1.  **新建檔案**：在左側**檔案總管**中右鍵選擇 `新建檔案 (New File)`。
2.  **命名規範 (重要)**：將其命名為 `ide_demo.py`。
    *   **準則**：在工程領域，檔案名應僅包含**小寫字母、數字與下劃線**。
    *   **禁忌**：**嚴禁使用空格**或**中文**命名代碼檔案，這會導致路徑解析失敗。
3.  **編寫內容**：在編輯器中輸入：
    ```python
    print("Hello AI IDE")
    ```
    *注意：此時你會發現代碼出現了不同的顏色（高亮），這說明 Python 延伸模組正在透過語法分析輔助你閱讀。*

### 4.2 執行方式對比
*   **手動執行**：在底部的 IDE 終端機中手動輸入 `python ide_demo.py`。
*   **整合執行**：確認你要執行的檔案處於**當前活動標籤頁 (Active Tab)**，點擊編輯器右上角的**執行按鈕**（三角形圖標）。

**工程師的習慣：保持終端機整潔**
當反覆執行代碼導致終端機輸出堆積如山時，可以在終端機輸入 `clear` (Mac) 或 `cls` (Windows) 並按下回車。這能清空舊輸出，讓你專注於當下的執行結果。
