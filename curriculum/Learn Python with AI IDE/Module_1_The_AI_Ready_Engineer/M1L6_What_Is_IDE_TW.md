## 1. Philosophy: 工具主權與 AI IDE 生態

### 1.1 主流 IDE 與 AI 增強件
在正式開始前，我們需要了解目前工程界主流的開發工具。雖然我們使用的是 Antigravity，但這些知識在其他工具中是通用的：

*   **VS Code**：當代軟體開發的「中央車站」。幾乎所有主流 IDE（包括 Antigravity）都是基於它的開源框架構建的。
*   **AI 原生 IDE**：如 **Cursor**、**Kirokiro** 以及字節跳動出品的 **Trae**。
*   **AI 增強外掛**：即使是普通的 IDE，也可以透過安裝外掛獲得 AI 輔助能力。例如 **GitHub Copilot** (基於 OpenAI 模型的代碼助手)、**Claude Code** 或 **Codex**。

### 1.2 為什麼需要「工具主權」 (Tool Sovereignty)？
在 AI 時代，學習 IDE 的重點不是點擊按鈕，而是掌握**協作的第一現場**。

*   **IDE 是互動的「唯一真值」**：雖然你可以透過網頁、終端機與 AI 互動，但代碼最終的落地、調試與運行，最主流且最穩健的場景依然是在 IDE 中。
*   **看清 AI 的每一個動作**：AI 修改了哪個檔案？在終端機執行了什麼指令？環境是否被污染？如果不掌握 IDE，你對 AI 的協作就像是「隔山打牛」。只有拿回工具主權，你才能在 AI 出錯時，依然擁有對工程的絕對控制力。

## 2. The Raw Logic：界面感知與「生存配置」

### 2.1 界面初始化 (The Initial Setup)
打開 Antigravity 後，我們需要將界面調整為最經典的「工程模式」。請確保以下三個板塊清晰可見：

1.  **左側欄 (Primary Side Bar)**：點擊左上角的方塊圖標。這是管理工程資產的區域。
2.  **隱藏 AI 面板 (Toggle Agent)**：暫時將其**關閉**。在學習階段，我們需要先看清工具的本原。
3.  **底部面板 (Panel)**：
    *   **Mac**: `Command + J`
    *   **Windows**: `Ctrl + J`
    這是你之前熟悉的終端機所在的區域。

現在，你應該看到一個穩定的佈局：**左側檔案總管**、**右上編輯區**、**下部控制區**。

### 2.2 核心圖標與延伸模組 (Extensions)
在左側邊欄，你會看到幾個核心功能圖標。作為 Python 工程師，最常用的是：
*   **檔案總管 (Explorer)**：管理當前項目的資料夾和檔案。
*   **延伸模組 (Extensions)**：安裝增強功能的「外掛商店」。

### 2.3 宿主與延伸模組的關係
IDE 就像一個精密的硬體支架（宿主），而**延伸模組 (Extensions)** 則是為它注入靈魂的功能模組。
1.  **Python 延伸模組**：這是必備組件。它賦予了 IDE 識別、執行、調試和智能提示 Python 代碼的能力。
2.  **中文語言包 (Traditional)**：安裝後重新啟動，界面將切換為繁體中文。

## 3. Instruction：關聯環境與項目

### 3.1 搬入項目資料夾
將我們在 M1L5 中辛苦打磨的「第一批移民」搬進 IDE。這也是我們在 L5 手動創建的那三個 `.py` 腳本檔案（`hello.py` 等）。
*   點擊菜單欄 `文件 (File)` -> `開啟資料夾 (Open Folder)`。
*   選擇該資料夾。此時，你可以在左側看到所有的檔案。

### 3.2 配置解釋器：插座與插頭 (Recall M1L4)
> **🔌 隱喻回調**
> 環境 `ai_course` 是我們的**專用電源插座**。現在，我們要為 IDE 這台設備接通電源。

1.  呼出**指令面板** (`Cmd/Ctrl + Shift + P`)。
2.  搜尋並選擇 `Python: Select Interpreter` (選擇解釋器)。
3.  在列表中選中帶有 `ai_course` 標識的那一項。
    *   *觀察*：IDE 右下角狀態欄應顯示 `ai_course`。

## 4. Practice：複刻與驗證

### 4.1 創建 IDE 演示腳本
1.  **創建檔案**：在左側**檔案總管**中右鍵點擊資料夾空白處，選擇 `新建檔案 (New File)`，命名為 `ide_demo.py`。
2.  **進入編輯狀態**：單擊剛創建的 `ide_demo.py`，你會在右上的**編輯區**看到一個空白的輸入視窗。
3.  **編寫代碼**：就像在記事本中輸入文字一樣，在視窗中輸入：
    ```python
    print("Hello AI IDE")
    ```
    *你會發現：輸入 `pri` 時會出現提示補全。這其實是 IDE 安裝了 Python 模組後的「詞典映射」功能，它能認出語法。未來我們加裝 AI 引擎後，它將不再只是認詞，而是能夠理解你的意圖。*

### 4.2 運行對比：手動 vs 一鍵
*   **方法 A (傳統終端機)**：在 IDE 底部的**終端機**裡，手動輸入 `python ide_demo.py`。
*   **方法 B (IDE 一鍵執行)**：點擊編輯器右上角的**執行按鈕**（三角形圖標）。

**硬核觀察**：點擊執行按鈕後，請死死盯著底部終端機。你會發現 IDE 幫你輸入了一行極長的指令。
**結論**：IDE 並不是魔法，它是一個**集成的自動化平臺**。除了運行代碼，未來你還將在這裡管理資料庫、管理 Git 倉庫。它是你作為工程師的**唯一完整工作站**。
